Work Order Module

The work order module is intended to help control the manufacturing process when assembling inventory. Features include:

  • Task driven work flow process
  • Sign-off ability for production workers and quality control
  • Task time tracking for job estimation
  • Revision control for process management
  • Ability to assemble into inventory (create a journal entry similar to Inventory -> Assemblies)
  • Uses Quantity of Allocation in inventory to identify lack of availability of assembly parts
  • Work Order form lists tasks, sign off requirements, required parts and sign off blocks.

How does it work? Work Orders are broken into a series of tasks. The tasks are combined to create a work order template for a given sku. The work order template is then executed with statistics, data and sign-offs recorded in a work order journal (not associated with the main journal of PhreeBooks)

Work Order Tasks - Overview

When building a product, the assembly process can be broken into a series of tasks. These tasks make up the building blocks of the work order. Click here for more detailed information regarding work order tasks.

Work Order Builder - Overview

Once the tasks for building an inventory item, the work order builder sequences the tasks required for assembling a specific product. The Work Order Builder only allows one sku for each template. Click here for more details on building a work order template. Once a work order template has been executed, it must be revised if the process is changed. This is for quality control.

Work Order Manager - Overview

The work order manager is the execution of a work order template. The work order manager tracks the assembly of an inventory item, allocates material and assembles item in the inventory database. Click here for more information on Work Order Manager.

An Example

We want to assemble a bicycle (must be in inventory as an assembly already). The steps required (very simply example) are:

  1. Pull parts from stock room.
  2. Assemble bicycle per assembly procedure.
  3. Inspect finished product.
  4. Record number of bicycles into inventory database.

Step 1: We need to create the tasks using the Work Order Task builder

  1. Click on Inventory -> Work Order Tasks.
  2. Enter information on the form Task Name : Pull Parts Description : Pull parts as listed on the work order form Department : Manufacturing (Set up in Company -> Module Administration -> Contacts -> Properties -> Maintain Departments tab
    Ref Documents & Ref Drawings : leave blank
    Task Time : 0.5 Hours
    Mfg Sign-off, QA Sign-off, Data Entry & ERP Entry : No
  3. Press the Save icon to save the task

Repeat the above procedure for the remaining 3 steps with the following differences:

Step 2. Change the task name and description, MFG Sign-off to Yes, Data Entry to Yes (for serial numbers)

Step 3: Change the task name and description. QA Sign-off to Yes.

Step 4: Change the task name and description. ERP Entry to Yes.

Step 2: We need to build a template.

  1. Click on Inventory -> Work Order Builder
  2. Click the New icon
  3. Enter the information on the form as follows: Title : Build Bicycle SKU : Select a SKU from the search pop-up icon. (the bicycle assembly needs to be in the Inventory database already), description is automatically filled in Use Allocation : Yes Ref Documents & Ref Drawings : blank
  4. On the first task line, click on the search icon and select Pull Parts from the pop-up.
  5. Click on the Add a line icon and repeat step 4 for the remaining three tasks.
  6. Click on the Save icon to save the work order template.

Step 3: We need to generate a work order to let manufacturing know to build a bicycle.

  1. Click on Inventory -> Work Order Manager
  2. Click the New icon
  3. Click the search icon next to the SKU field and select the Build Bicycle work order. The title and SKU are filled into the form along with the task list.
  4. Enter the number of bicycles to build, we’ll build 2 for now.
  5. Enter any comments, special instructions, etc. onto the Notes field.
  6. Click on the Print icon to save the work order and print the Work Order form. The screen will return to the Work Order Manager and should display our work order with open status. This will allocate the parts required to assemble the bicycle and verify stock is available.

Step 4: Let’s build a bicycle

  1. Select the work order from the Work Order Manager screen. this will open the build page highlighting the step in process.
  2. When step 1 has been completed, click on the Update button on the right of the screen. Step 1 will be posted and step two will be highlighted.
  3. When step 2 has been completed, enter the assemblers user name and password and press the update button. remember in task Assemble bicycle, we required manufacturing sign-off to track who assembled the bicycle.
  4. When step 3 has been completed, enter QA’s user name and password and press the Update button. This will post step 3 and highlight step 4.
  5. Press update on step 4 to complete the work order, close it out, assemble 2 bicycles in inventory (assuming parts are available) and un-allocate the parts (as they have been used to build the bicycle).
